# Accessibility

## Current Baseline

- Commands are available via command palette and context menus.
- Status and errors are communicated in text in addition to icons.
- Webviews should support keyboard navigation patterns provided by standard HTML controls.

## Requirements

1. Action labels must be explicit and unambiguous.
2. Critical state changes should not rely only on color.
3. Keyboard-first command invocation remains a first-class interaction path.
