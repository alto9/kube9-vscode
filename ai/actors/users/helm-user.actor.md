# Helm User

## Overview

A developer who uses Helm charts to deploy, configure, and manage applications on Kubernetes clusters. They work with Helm repositories, discover charts, customize values, and manage release lifecycles.

## Responsibilities

- Manage Helm repositories (add, update, remove)
- Discover and search for Helm charts across repositories
- Install charts with customized values
- Monitor installed releases and their status
- Upgrade releases when new versions are available
- Rollback releases when upgrades fail
- Uninstall releases when applications are no longer needed
- Manage chart values and configurations

## Characteristics

- Has Helm CLI installed on their machine
- Familiar with Helm concepts (charts, releases, repositories, values)
- May manage multiple Kubernetes clusters
- Prefers visual tools over CLI for routine operations
- Values quick access to chart documentation and metadata
- Needs visibility into release status and history

## Examples

A Helm user installing the Kube9 Operator by browsing the featured charts section, clicking "Install Now", optionally entering an API key, and confirming installation with pre-configured values. After installation, they monitor the release status and upgrade it when new versions become available.
